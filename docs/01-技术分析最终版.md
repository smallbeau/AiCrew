## 1. 项目概述

**灵犀Ai（超级员工）** 是一款基于 Electron 的企业级 AI 社交媒体营销、获客和员工管理桌面应用。集成了多个社交媒体平台的自动化操作、企业微信管理、AI 内容创作、数字人视频生成等功能。

### 核心定位

- **目标用户**: 企业营销团队、私域运营人员、HR 招聘人员
- **核心价值**: 通过 AI 和自动化技术，提升社交媒体运营、客户获取、员工管理效率
- **支持平台**: 抖音、小红书、快手、企业微信、Boss直聘

### 主要功能类别

| 类别 | 功能数 | 说明 |
|------|--------|------|
| AI 内容创作 | 8 | 短视频矩阵、多场景剪辑、素材管理 |
| 数字人短视频 | 5 | 声音定制、形象定制、视频生成 |
| Sora2 AI 视频 | 2 | 文生图、文生视频 |
| AI 拓客 | 2 | 智能获客、地图获客 |
| AI 个微 | 4+ | 聚合聊天、标签管理、关键词回复 |
| AI 企微 | 12+ | 完整企业微信管理功能 |
| AI 人事 | 6 | Boss直聘管理、候选人沟通 |
| AI 法务 | 3 | 法务咨询、检索、合同审查 |
| 企业智库 | 4 | 智能体、知识库、话术管理 |
| 企业管理 | 2 | 部门管理、员工管理 |
| 数据罗盘 | 2 | 算力消耗、员工数据 |
| 培训内容 | 2 | 培训课堂、员工培训 |

### 核心特性

| 特性 | 描述 |
|------|------|
| **桌面外壳** | Electron + Node.js 后端 |
| **前端UI** | Vue.js 2.x + Element UI |
| **自动化引擎** | Playwright (Chromium无头浏览器) |
| **本地存储** | SQLite (better-sqlite3) |
| **后端API** | 远程API + 本地Express服务 |
| **多平台支持** | 抖音/小红书/快手/Boss直聘/企业微信 |

### 远程API域名

| 域名 | 用途 |
|------|------|
| `https://szr.huabeiwangluo.com` | 主业务API（BASE_API） |
| `https://api-m.minicps.com` | 远程配置/更新接口（config.conf） |
| `crm.huabeiwangluo.com` | CRM及更新下载 |
| `lingxiai.huabeiwangluo.com` | 灵犀AI专属服务 |

### 3.2 进程通信模型

```
渲染进程 (Vue.js)
    │
    ├── ipcRenderer.send() / ipcRenderer.invoke()
    │
    ▼
主进程 (Node.js)
    │
    ├── ipcMain.on() / ipcMain.handle()
    │
    ├── Express Server (本地 HTTP, 端口 25565)
    │       │
    │       ├── /api/health              → 健康检查
    │       ├── /api/wecom/callback      → 企微回调
    │       ├── /api/wecom/msg/sync      → 同步消息
    │       └── /call_back (端口 18060)  → 回调处理
    │
    ├── WeCom Server (端口 8060)
    │       │
    │       └── TCP 端口 9090
    │
    └── 远程 API 调用
            │
            └── https://api-m.minicps.com / https://szr.huabeiwangluo.com
```

## 7. 数据库设计（完整DDL）

### 7.1 SQLite 数据库

**数据库文件**: `super_employee.db`
**存储位置**: `app.getPath('userData')` 或项目根目录的 `../../../../super_employee.db`
**WAL模式**: `PRAGMA journal_mode = WAL`

### 7.2 all_customers（客户表）

```sql
CREATE TABLE IF NOT EXISTS all_customers (
    id TEXT PRIMARY KEY,           -- 唯一 ID (用户sec_uid/uid/comment_id等)
    account TEXT,                  -- 所属账号
    platform INTEGER,              -- 平台：1=抖音，2=小红书，3=快手
    category INTEGER,              -- 获客方式：1=对标账号, 2=视频评论, 3=关键词, 4=直播
    nickname TEXT,                 -- 用户昵称
    content TEXT,                  -- 评论/私信内容
    create_time INTEGER,           -- 创建时间（毫秒时间戳）
    has_intent INTEGER DEFAULT 0,  -- 是否有意向：1=有, 0=无
    raw_data TEXT                  -- 原始 JSON 字符串
);

-- 索引
CREATE INDEX IF NOT EXISTS idx_platform_cat ON all_customers (platform, category);
CREATE INDEX IF NOT EXISTS idx_account ON all_customers (account);
```

**字段详解**:

| 字段 | 类型 | 说明 |
|------|------|------|
| id | TEXT PK | 用户唯一标识，优先使用 sec_uid > uid > comment_id > 随机生成 |
| account | TEXT | 执行获客操作的归属账号名称 |
| platform | INTEGER | 1=抖音, 2=小红书, 3=快手 |
| category | INTEGER | 获客渠道分类 |
| nickname | TEXT | 用户昵称 |
| content | TEXT | 评论或消息内容 |
| create_time | INTEGER | 时间戳（毫秒级），10位时间戳自动×1000 |
| has_intent | INTEGER | 关键词匹配标记，1=含购买意向关键词 |
| raw_data | TEXT | 原始JSON数据完整保留 |

### 7.3 operation_logs（操作日志表）

```sql
CREATE TABLE IF NOT EXISTS operation_logs (
    id INTEGER PRIMARY KEY AUTOINCREMENT,  -- 自增ID
    account TEXT,                          -- 所属账号
    platform INTEGER,                      -- 平台：1=抖音, 2=小红书, 3=快手
    category INTEGER,                      -- 获客方式：1-4
    type TEXT,                             -- 操作类型：'1'=关注, '2'=私信, '3'=回复, '4'=点赞
    create_time INTEGER,                   -- 创建时间（毫秒）
    raw_data TEXT                          -- 完整详情 JSON
);

-- 索引
CREATE INDEX IF NOT EXISTS idx_log_query ON operation_logs (account, create_time);
```

### 7.4 hk_fans（粉丝表）

```sql
CREATE TABLE IF NOT EXISTS hk_fans (
    id TEXT PRIMARY KEY,               -- 外层唯一 ID (sec_uid)
    task_id TEXT,                      -- 关联任务 ID
    platform INTEGER,                  -- 平台：1=抖音, 2=小红书
    account TEXT,                      -- 所属账号
    nickname TEXT,                     -- 用户昵称
    msg_status INTEGER DEFAULT 0,      -- 私信状态：0=未发送, 1=已发送
    follow_status INTEGER DEFAULT 0,   -- 关注状态：0=未关注, 1=已关注

    user TEXT                          -- 粉丝完整 JSON 字符串
```

### 7.6 历史数据迁移

代码中包含从旧版本文件系统迁移到新数据库的逻辑：

```
旧版存储路径:
  {userData}/all_comment/index.json              → 操作日志旧数据
  {userData}/comments/{platform}/{account}/{id}.json   → 评论旧数据

迁移后:
  all_customers 表 + operation_logs 表
```

### 10.2 企业微信服务 (wecom/)

#### localhost_serve.exe

| 属性 | 值 |
|------|-----|
| 端口 | 8060 |
| TCP端口 | 9090 |
| 回调URL | `http://127.0.0.1:18060/call_back` |
| 角色 | 企业微信消息代理服务 |

**功能描述**:
- 作为企业微信与主应用之间的 HTTP 代理
- 长轮询机制获取企业微信消息
- 接收来自企微的消息回调
- 转发消息到主应用IPC处理
- 设备授权与状态管理

**通信方式**:

```
  ┌──────────────┐    HTTP长轮询    ┌──────────────┐    IPC    ┌──────────┐
  │ 企业微信服务 │ ←─────────────→ │ localhost_   │ ←──────→ │ Electron │
  │ 器           │                  │ serve.exe   │          │ 主进程   │
  └──────────────┘   回调Webhook    │ (端口8060)   │          └──────────┘
                                     └──────────────┘
                                          ↕ TCP (9090)
                                     ┌──────────────┐
                                     │ voice_agent  │
                                     │ .exe         │
                                     └──────────────┘
```

#### voice_agent.exe（语音助手服务）

- 语音助手服务

## 13. 部署与更新机制

### 13.1 双更新策略

#### 1. electron-updater 自动更新
- 标准 Electron 自动更新机制
- 通过 GitHub Releases 或自定义服务器分发
- 用户可选择是否更新

```javascript
const { autoUpdater } = require('electron-updater');

// 配置
autoUpdater.setFeedURL({
    provider: 'generic',
    url: 'https://crm.huabeiwangluo.com/download/exe/'
});

// 更新检查
ipcMain.handle('check-update', (event) => {
    autoUpdater.checkForUpdates();
});

// 事件监听
autoUpdater.on('update-available', (info) => {
    // 通知用户有更新可用
});
autoUpdater.on('download-progress', (progress) => {
    // 下载进度
});
autoUpdater.on('update-downloaded', () => {
    // 下载完成
});
```

#### 2. 热 ASAR 更新
- 下载新的 ASAR 文件
- 替换当前 ASAR 文件
- 无需完全重启应用
- 更新配置: `set-version-config`

### 13.2 更新流程

```
检查更新 → 下载更新 → 确认更新 → 安装更新 → 重启应用
    │           │           │           │           │
    ▼           ▼           ▼           ▼           ▼
check-update  download   confirm    download    execute
              -update    -update    -install    _hot_update
```
