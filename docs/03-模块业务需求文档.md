## 1. 登录模块

### 1.1 模块概述

登录模块是系统的入口，负责用户身份认证、会话管理及多设备登录控制。支持三种登录方式（密码登录、微信扫码、手机验证码），确保用户能够安全、便捷地访问系统。登录后生成的 Token 有效期为 7 天，到期后自动刷新或要求重新登录。

### 1.2 用户角色

| 角色 | 说明 |
|------|------|
| 未注册访客 | 可通过注册入口创建账号 |
| 注册用户 | 已激活账号的普通用户，可正常登录访问所有功能 |
| 被封禁用户 | 因违规被管理员封禁，登录时提示封禁原因 |

### 1.3 功能清单

| 功能点 | 优先级 | 说明 |
|--------|--------|------|
| 用户名+密码登录 | P0 | 支持邮箱/手机号作为用户名，密码需经过加盐哈希传输 |
| 微信扫码登录 | P0 | 通过微信开放平台扫码授权，首次扫码自动绑定已有账号或创建新账号 |
| 手机验证码登录 | P0 | 输入手机号后发送 6 位验证码，60 秒内有效，60 秒后可重新发送 |
| 用户注册 | P0 | 填写手机号、密码（8-20位含字母+数字）、获取验证码完成注册 |
| 忘记密码 | P1 | 通过手机验证码重置密码，重置后所有设备下线 |
| Token 自动刷新 | P0 | 请求 401 时自动携带 refreshToken 获取新 accessToken |
| 多设备登录管理 | P1 | 展示当前登录设备列表（设备名、最后活跃时间），支持远程下线 |
| 登录日志 | P2 | 记录每次登录的时间、IP、设备、登录方式，支持用户查看 |
| 账号注销 | P2 | 用户主动注销账号，需二次确认和身份验证 |

### 1.4 业务流程

**登录流程：**

1. 用户打开登录页，默认展示密码登录 Tab
2. 用户输入账号密码，点击「登录」
3. 前端对密码做 SHA256 哈希后传输到后端
4. 后端校验账号密码，校验通过后生成 accessToken（7 天有效）+ refreshToken（30 天有效）
5. 前端将 Token 存入 localStorage，跳转至 Dashboard 页面
6. 校验失败时，根据错误类型提示：账号不存在 / 密码错误 / 账号被封禁（附原因）/ 登录过于频繁（60 秒后重试）

**微信扫码登录流程：**

1. 用户点击「微信扫码登录」，前端请求后端获取微信扫码 URL 和 sceneId
2. 弹出二维码弹窗，用户使用微信扫码
3. 微信回调通知后端，后端绑定用户信息
4. 前端通过轮询 sceneId 获取登录结果，成功后跳转 Dashboard

**手机验证码登录流程：**

1. 用户输入手机号，点击「获取验证码」
2. 后端校验手机号格式，发送 6 位验证码短信
3. 前端开始 60 秒倒计时，倒计时结束后按钮恢复可点击
4. 用户输入验证码，点击「登录」
5. 后端校验验证码有效期和正确性，校验通过后登录成功

### 1.5 数据定义

| 字段名 | 类型 | 说明 | 示例 |
|--------|------|------|------|
| userId | String | 用户唯一 ID | "u_20260615001" |
| username | String | 用户名 | "zhangsan" |
| phone | String | 绑定的手机号 | "13800138000" |
| wechatOpenId | String | 微信开放平台 OpenID | "oX8_0..." |
| passwordHash | String | 加盐后的密码哈希 | "$2b$10$..." |
| status | Enum | 账号状态：active/banned/inactive | "active" |
| banReason | String | 封禁原因（被封禁时返回） | "违规发布广告" |
| accessToken | String | 访问令牌 | "eyJhbG..." |
| refreshToken | String | 刷新令牌 | "eyJhbG..." |
| tokenExpireAt | DateTime | Token 过期时间 | "2026-06-22T13:40:00+08:00" |
| deviceInfo | JSON | 设备指纹信息 | {"os":"Windows","browser":"Chrome","deviceId":"abc123"} |

### 1.6 交互规则

- **登录按钮状态**：加载中显示旋转动画并禁用按钮，文案变为「登录中...」
- **表单校验**：前端实时校验邮箱格式、密码长度，错误提示红色文字显示在输入框下方
- **验证码发送**：点击后按钮禁用并显示倒计时「60s」，倒计时结束恢复为「重新获取」
- **微信登录弹窗**：320px × 480px 居中弹窗，含二维码和「打开微信扫码」提示文字，扫码成功后自动关闭
- **登录错误 Toast**：顶部居中显示，3 秒后自动消失
- **Token 过期**：后端返回 401，前端拦截器自动使用 refreshToken 重试，失败则清除 Token 并跳转登录页
- **多设备下线**：在「设备管理」弹窗中点击「下线」，确认后该设备的 Token 立即失效

### 1.7 与其他模块的关系

| 关联模块 | 交互说明 |
|----------|----------|
| 系统设置 | 登录后在系统设置中可修改密码、更换绑定手机号 |
| 算力消耗 | 登录后系统开始消耗算力，未登录状态下所有功能不可用 |

### 1.8 验收标准

- [ ] 三种登录方式均可正常登录，跳转至 Dashboard 不超过 3 秒
- [ ] 密码登录连续 5 次失败后，账号锁定 15 分钟
- [ ] 微信扫码登录二维码有效期为 5 分钟，过期后展示「二维码已过期，请刷新」
- [ ] 手机验证码 60 秒内重复使用无效，验证码正确但已超过 60 秒时提示失效
- [ ] Token 7 天有效，到期后自动刷新，刷新失败跳转登录页
- [ ] 用户可在「设备管理」中看到所有登录设备，可远程下线任意设备
- [ ] 被封禁用户登录时展示封禁原因弹窗，无法进入系统

## 2. 数据总览 Dashboard

### 2.1 模块概述

Dashboard 是用户登录后默认进入的核心数据看板，集中展示平台运营的关键指标（今日获客数、活跃设备数、算力余额、转化率），并提供趋势图表、平台分布分析和实时动态信息流。帮助用户快速掌握整体运营状况，并可通过快捷入口直达高频功能。

### 2.2 用户角色

| 角色 | 说明 |
|------|------|
| 管理员 | 查看全局数据，掌握团队整体获客情况 |
| 运营人员 | 日常查看数据趋势，发现问题及时调整策略 |
| 普通用户 | 查看自己的获客数据，了解线索转化情况 |

### 2.3 功能清单

| 功能点 | 优先级 | 说明 |
|--------|--------|------|
| 核心指标卡片 | P0 | 展示今日获客数、活跃设备数、算力余额、转化率四个指标，含环比变化（↑↓） |
| 7 天/30 天获客趋势图 | P0 | 折线图，支持在 7 天和 30 天之间切换，展示每日获客量 |
| 平台分布饼图 | P0 | 展示抖音/小红书/快手/视频号各平台线索数量占比 |
| 实时动态流 | P0 | 按时间倒序展示最新的客户线索和系统操作记录，自动滚动更新 |
| 快捷入口 | P0 | 卡片式快捷按钮：新建短视频任务、创建剪辑、智能获客、查看线索 |
| 数据刷新控制 | P1 | 手动刷新按钮 + 自动刷新开关（每 30 秒刷新） |
| 指标说明 Tooltip | P1 | 每个指标卡片右上角的「?」图标，悬停展示指标计算方式 |
| 数据导出 | P2 | 将趋势图数据导出为 CSV 文件 |
| 定制布局 | P2 | 允许用户拖拽调整卡片排序和显示/隐藏 |

### 2.4 业务流程

1. 用户登录后自动跳转到 Dashboard
2. 页面加载时并行请求以下数据：
   - 核心指标（今日获客数、活跃设备数、算力余额、转化率）
   - 7 天获客趋势数据
   - 平台分布数据
   - 最近 20 条实时动态
3. 数据加载完毕后渲染各组件，展示加载动画骨架屏
4. 用户可以切换趋势图的统计周期（7 天/30 天），切换时重新请求对应数据
5. 实时动态流每 10 秒拉取新数据，追加到列表顶部，列表最多保留 50 条
6. 用户点击快捷入口跳转到对应功能页
7. 页面关闭时停止实时动态流轮询

### 2.5 数据定义

| 字段名 | 类型 | 说明 | 示例 |
|--------|------|------|------|
| todayLeads | Number | 今日新增线索数 | 47 |
| leadsChangeRate | String | 线索环比变化百分比 | "+12.5%" |
| activeDevices | Number | 当前活跃设备数 | 8 |
| devicesChangeRate | String | 设备环比变化 | "-3.2%" |
| computeBalance | Number | 算力余额（点数） | 3850 |
| balanceChangeRate | String | 算力余额变化 | "-5.0%" |
| conversionRate | Number | 线索到客户的转化率百分比 | 23.5 |
| rateChangeRate | String | 转化率环比变化 | "+1.8%" |
| trendData | Array | 趋势图数据点数组 | [{"date":"06-08","count":32},...] |
| platformDistribution | Array | 各平台线索分布 | [{"platform":"抖音","count":328,"percent":45.2},...] |
| realtimeFeed | Array | 实时动态列表 | [{"type":"lead","content":"用户xxx提交了表单","time":"13:35:20"},...] |

### 2.6 交互规则

- **骨架屏**：页面加载时每个卡片区域展示灰色骨架屏，抖动动画
- **指标卡片**：数字用大号加粗字体（32px），环比变化用绿色（上升）或红色（下降）带箭头图标
- **趋势图**：使用 ECharts 折线图，X 轴为日期，Y 轴为获客数，鼠标悬停显示具体数值 Tooltip
- **饼图**：不同平台使用不同颜色（抖音红、小红书橙、快手黄、视频号绿），点击扇区可下钻查看该平台详情
- **实时动态流**：新条目从顶部滑入动画，不同类型的动态用不同图标（线索=👤、操作=⚙️）
- **快捷入口**：四个卡片，每个卡片包含图标、名称和简短描述，点击后路由跳转
- **数据为空**：展示空状态插画和「暂无数据，开始你的第一个获客任务吧」引导文案
- **加载失败**：展示错误插画和「数据加载失败」+「重新加载」按钮

### 2.7 与其他模块的关系

| 关联模块 | 交互说明 |
|----------|----------|
| 短视频矩阵 | Dashboard 展示短视频任务的获客数据，点击「新建任务」快捷入口跳转到短视频矩阵页 |
| 智能获客 | Dashboard 展示智能获客的线索统计和转化率 |
| 地图获客 | Dashboard 中的平台分布可下钻查看地图获客的线索明细 |
| 算力消耗 | Dashboard 展示算力余额，支持点击跳转到算力中心充值 |
| 实时动态 | 所有模块产生的线索和操作记录都会推送到 Dashboard 实时动态 |

### 2.8 验收标准

- [ ] 首次加载页面速度不超过 3 秒（含所有接口请求）
- [ ] 四个核心指标数据准确，环比变化计算正确
- [ ] 趋势图可正常切换 7 天/30 天，切换请求不超过 1 秒
- [ ] 平台分布饼图数据占比加总为 100%
- [ ] 实时动态每 10 秒自动更新，不重复展示已展示的条目
- [ ] 所有卡片均支持空状态、加载状态和错误状态

## 3. 短视频矩阵

### 3.1 模块概述

短视频矩阵模块是 AiCrew 的核心功能之一，支持用户在抖音、小红书、快手、视频号等主流短视频平台进行批量内容发布和管理。用户可以创建发布任务、管理多平台账号、查看视频数据统计、实现「一次制作，多平台分发」的高效运营模式。

### 3.2 用户角色

| 角色 | 说明 |
|------|------|
| 内容运营 | 创建和管理发布任务，监控视频表现数据 |
| 账号管理员 | 绑定/解绑平台账号，管理授权状态 |
| 团队负责人 | 查看团队整体发布数据和效果 |

### 3.3 功能清单

| 功能点 | 优先级 | 说明 |
|--------|--------|------|
| 任务创建 | P0 | 选择发布平台（抖音/小红书/快手/视频号，支持多选）、上传视频、填写标题/描述/标签 |
| 发布计划设置 | P0 | 立即发布 / 定时发布（精确到分钟） / 分批发布（设置各平台发布时间间隔） |
| 批量发布 | P0 | 一个视频同时发布到多个已绑定的平台账号 |
| 任务管理列表 | P0 | 列表展示所有任务（状态/平台/标题/发布时间/数据概览），支持筛选和搜索 |
| 任务启动/暂停 | P0 | 暂停任务可暂停定时发布计划，启动后继续执行 |
| 任务删除 | P1 | 删除未发布或已完成的定时任务，已发布的视频不能删除 |
| 查看任务状态 | P0 | 每个任务展示当前状态：待发布/发布中/已发布/发布失败/已暂停 |
| 数据统计 | P0 | 每个视频的播放量、点赞数、评论数、分享数、转化数 |
| 多维度数据筛选 | P1 | 按平台、日期范围、任务状态筛选视频数据 |
| 账号绑定 | P0 | 通过 OAuth 授权绑定平台账号，支持绑定多个账号 |
| 账号解绑 | P0 | 解除已绑定的平台账号，解绑后该账号的任务不可执行 |
| 账号授权过期提醒 | P1 | 账号授权即将过期（7天内）时展示提醒，支持重新授权 |
| 视频素材库 | P2 | 上传的视频自动存入素材库，支持重复使用 |
| 批量数据导出 | P2 | 将选定任务的数据统计导出为 Excel |

### 3.4 业务流程

**创建发布任务流程：**

1. 用户进入短视频矩阵页面，点击「新建任务」按钮
2. 弹出任务创建向导，步骤一：选择平台
   - 展示已绑定的平台账号列表，支持多选
   - 未绑定账号的平台显示「去绑定」按钮
3. 步骤二：上传视频
   - 支持拖拽上传或点击选择文件，单文件最大 500MB
   - 上传时展示进度条，支持同时上传多个视频
   - 视频上传完成后自动进行格式校验和转码
4. 步骤三：填写信息
   - 标题（各平台字符限制不同，实时提示）
   - 描述/文案（支持 AI 辅助生成文案）
   - 标签/话题（支持从热门推荐中选取）
   - 封面图（可选，自动截取视频帧或自定义上传）
5. 步骤四：设置发布计划
   - 立即发布 / 定时发布（选择日期时间）/ 分批发布（设置各平台时间间隔）
6. 点击「确认发布」，返回任务列表，新任务出现在列表顶部

**账号绑定流程：**

1. 用户点击「账号管理」进入账号列表页
2. 点击「绑定账号」，选择平台类型
3. 跳转到平台 OAuth 授权页（弹窗打开），用户授权后获取 accessToken
4. 授权成功后，账号自动添加至列表，展示平台名称、头像、昵称、授权有效期
5. 若授权失败，提示具体原因并提供重试按钮

### 3.5 数据定义

| 字段名 | 类型 | 说明 | 示例 |
|--------|------|------|------|
| taskId | String | 任务唯一 ID | "svt_20260615001" |
| title | String | 视频标题 | "产品使用教程第3期" |
| platforms | Array | 发布的平台列表 | ["douyin","xiaohongshu","kuaishou"] |
| accountIds | Array | 使用的账号 ID 列表 | ["acc_dy_001","acc_xhs_001"] |
| videoUrl | String | 视频文件 URL | "https://cdn.icrew.com/videos/xxx.mp4" |
| coverUrl | String | 封面图 URL | "https://cdn.icrew.com/covers/xxx.jpg" |
| description | String | 视频描述/文案 | "保姆级教程，3分钟学会..." |
| tags | Array | 标签/话题列表 | ["#教程","#产品","#效率工具"] |
| scheduleType | Enum | 发布计划类型：immediate/scheduled/batch | "scheduled" |
| publishTime | DateTime | 计划发布时间 | "2026-06-16T10:00:00+08:00" |
| batchInterval | Number | 分批发布的间隔时间（分钟） | 30 |
| status | Enum | 任务状态：pending/publishing/published/failed/paused | "published" |
| failReason | String | 发布失败原因 | "抖音API返回1062:标题含敏感词" |
| stats | JSON | 统计数据 | {"views":15230,"likes":892,"comments":156,"shares":234,"conversions":18} |
| createdAt | DateTime | 任务创建时间 | "2026-06-14T14:30:00+08:00" |

### 3.6 交互规则

- **任务创建向导**：分步式弹窗，底部有「上一步」「下一步」「取消」按钮，步骤数字标识当前进度（1/4、2/4...）
- **视频上传**：拖拽区域有虚线边框和上传图标，上传中显示进度百分比，支持取消上传
- **平台选择**：复选框卡片样式，卡片上展示平台 Logo 和绑定的账号昵称
- **定时发布**：时间选择器精确到分钟，过去的日期时间不可选，自动验证定时时间在当前时间 5 分钟后

## 12. 算力消耗

### 12.4 业务流程

**购买套餐流程：**

1. 用户进入算力消耗页面，在「套餐购买」区域看到三个套餐卡片
2. 每个卡片展示：套餐名称、价格、包含算力点数、功能权益列表、**推荐标签（热销/最值）**
3. 用户点击「立即购买」选择套餐
4. 弹出确认订单弹窗：套餐名称、价格、支付方式（微信/支付宝）
5. 用户选择支付方式，点击「确认支付」
6. 弹出支付二维码（或跳转支付页面），用户完成支付
7. 支付成功后显示「购买成功」，算力余额立即更新，展示「+3000 点」动画提示

**算力消耗结算流程：**

1. 用户执行任一消耗算力的操作（AI 对话、视频导出、智能剪辑等）
2. 系统根据操作类型和资源消耗量计算消耗点数
3. 从用户余额中扣除对应点数
4. 在消耗记录中立即生成一条记录
5. 若余额不足以完成操作，弹出提示「算力不足，请充值」，并提供快捷跳转按钮

### 13.4 业务流程

**修改密码流程：**

1. 用户进入「系统设置」→「账号设置」→ 点击「修改密码」
2. 弹窗表单：输入旧密码 → 输入新密码 → 确认新密码
3. 前端实时校验：新密码 8-20 位，必须包含字母和数字
4. 提交后端校验旧密码正确性
5. 旧密码正确 → 更新密码成功 → 弹出提示「密码修改成功，请重新登录」
6. 所有设备 Token 失效，强制跳转登录页
7. 旧密码错误 → 提示「旧密码不正确，剩余 X 次尝试机会」

**API 密钥配置流程：**

1. 用户进入「系统设置」→「API 配置」
2. 查看已生成的 API Key 和 Secret Key（默认系统自动生成）
3. 点击「重置密钥」，弹窗确认「重置后原有密钥将立即失效，确认重置？」
4. 确认后生成新的密钥对，展示在新的卡片中
5. 用户勾选「我已保存新的密钥」后关闭卡片
6. 查看使用说明：示例代码（curl / Python / Node.js 等）
